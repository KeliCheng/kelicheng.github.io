使用OAuthSwift完成微博OAuth2.0授权

|

昨天关于新浪的授权鼓捣了一天才弄好，所以写一个这个过程。主要用到的是OAuthSwift（一个适用于iOS／macOS等的OAuth库，支持1.0和2.0）
另外关于整个授权的过程，Practical Guide for using Sina Weibo’s API 也很有帮助。

  1. 新浪开发者，申请App，填写认证和App资料（略）
  2. 在project中导入OAuthSwift库（略）
  3. 授权 doOAuthWeibo
  1. 关于callback URL
    因为微博只接受http格式的URL，所以我在自己博客新建了一个页面作为callback URL（http://kelicheng.github.io/simple-sender/）
    在这个页面放一段代码重新定向到App中。

此外还要在project中设置URL Schemes:
Targets -> (YourAPP) -> Info -> URL Types -> Add -> URL Schemes = YourAppName

OAuthSwift的作者还给出了Ruby和PHP的代码示例，以及另一种使用内置浏览器授权的解决方案。详见：https://github.com/OAuthSwift/OAuthSwift/wiki/API-with-only-HTTP-scheme-into-callback-URL

Pointing Hover Domain | Hexo-GithubPages-GoDaddy

|

Domain: example.com
Blog: build with Hexo, hosted on GithubPages (example.github.io)

  1. create CNAME file uder the “source” directory (not the root dir)
    within the CNAME file, write “http://example.com”
    save it, generate (hexo g) and deploy (hexo d) to Github Pages

  2. go to GoDaddy DNS Manager of domain “example.com”
    add new records:
    • Hostname: @, Record Type: A, Value: 192.30.252.153
    • Hostname: @, Record Type: A, Value: 192.30.252.154
    • Hostname: www, Record Type: CNAME, Value: example.github.io

cs193p | AxesDrawer to Swift 3.0

|

Issue: cs193p Assignment 3, the simulator is blank when calling AxesDrawer

Intro: I’m taking the Standford University’s cs193p course, Developing iOS 9 Apps with Swift, through iTunesU. The course is for Spring 2016, but Swift has updated to 3.0 in October. Thus parts of the source code provided need to be updated manually. Most of this correction can be done with Xcode’s warning and suggestion, but there are still some tricky pieces.

I was working on Assignment III, Graphing Calculator. It provided the AxesDrawer.swift to help draw the axes. Here is what I did to fix some issues when converting it to Swift 3.0.

  1. put the file in the Calculator folder

  2. fix most warning with Xcode suggestions

  3. line 97
    let label = formatter.stringFromNumber((origin.x-bbox.minX)/pointsPerUnit)!
    the warning shows “no / candidates produce the expected contextual”
    replace it with:
    let label = formatter.string(from: (origin.x-bbox.minX)/pointsPerUnit as NSNumber)!

  4. call AxesDrawer in GraphView

Unfortunately, it doesn’t work. The simulator is blank, even the back arrow is missing. The problem is because drawAxesInRect is calling drawHashmarksInRect, and the while loop at the end of drawHashmarksInRect is causing an infinite loop.

  1. Through looking at the code, I found that the infinite loop is because the bbox was not updated with insetBy.
    So, fixing the last line of drawHashmarksInRect with:
    bbox = bbox.insetBy(dx: -pointsPerHashmark, dy: -pointsPerHashmark)

Finally, the AxesDrawer works perfectly.

I’m writing this post because I didn’t find a similar situation through searching. If you are also taking this Spring course, or have some trouble converting a file to Swift 3.0, or facing a blank simulator, hope this will give you some idea. Thanks.
